# PostgresPlanShow1C

Инструмент для анализа планов запросов PostgreSQL в реальном времени.

## Возможности

- **Настройка логирования** — автоматическое добавление параметров `auto_explain` в `postgresql.conf`
- **Мониторинг логов** — отслеживание и извлечение планов запросов из лог-файлов PostgreSQL
- **Отправка на визуализаторы** — загрузка планов на внешние сайты для визуального анализа
- **Управление сервисом** — перезапуск PostgreSQL с правами администратора
- **Поддержка форматов** — текстовый и JSON форматы планов запросов

## Установка

не требуется

## Использование

1. Запустите приложение
2. Укажите путь к `postgresql.conf`
3. Укажите путь к лог-файлу PostgreSQL
4. Нажмите "Включить" для добавления настроек логирования
5. Нажмите "Перезапустить postgresql" для применения изменений
6. Нажмите "Начать отслеживание" для мониторинга планов
7. Выберите план из таблицы и нажмите "Отправить на сайт" для визуализации

## Параметры

- **Минимальное время выполнения** — минимальная длительность запроса в мс для логирования (по умолчанию: 10000)
- **Формат JSON** — собирать планы в формате JSON (рекомендуется для визуализаторов)
- **Периодичность** — интервал проверки лог-файла в секундах (по умолчанию: 10)
- **Количество планов** — максимальное число планов для отображения (по умолчанию: 5)

## Конфигурация PostgreSQL

Приложение добавляет следующие параметры в `postgresql.conf`:

```
shared_preload_libraries = 'auto_explain'
auto_explain.log_min_duration = {}
auto_explain.log_format = 'json'  # или 'text'
auto_explain.log_analyze = on
auto_explain.log_buffers = on
auto_explain.log_timing = on
auto_explain.log_verbose = on
auto_explain.log_nested_statements = on
```

## Требования к правам

- Для настройки логирования и перезапуска PostgreSQL требуются права администратора
- На Windows приложение должно быть запущено от имени администратора
- На Linux/macOS требуются права sudo для управления сервисом PostgreSQL

## Поддерживаемые платформы
